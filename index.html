What is an API!?!

The technical definition is, an Application Progamming Interface (API), is a set of routines, protocols, and tools for building software applications. An API specifies how software components should interact.
To new developers this definition might not explain the real power of an API. While below I will try my best to abstractly explain the usefullness of an API, there are good websites out there that go more in-depth. The READ MORE button goes to a good website that I found useful.

The Waiter...

The best analogy to convey the abstract concept of what an API is and what it does, is likening it to the process of ordering food at a restaurant. It is about dinner time and you are hungry but you do not want to cook a meal from scratch. So you begin to look for restaurants that will satisfy your hunger. But you want a particular food that can only satisfy your particular hunger. After finding the right restaurant you sit down and begin looking at the menu. After you have chosen what you wanted the Waiter asks you for your order, the waiter writes it down, and then takes the order to the kitchen. After sometime the waiter reappears with your food.

While this story sounds obvious it shows why API's were created and what API's do. The hungry patron is the developer who has a specific problem. The restaurant is the API provider who creates the API for the developer. The developer goes to the API provider (hungry patron and restaurant) and looks at the menu to choose from. The developer finds a what they want and orders it through the waiter. The Waiter symbolizes the API. It takes the developers information and sends it to the system. Hopefully getting back the food/information wanted.

BreweryDB

Now that you have a a small grasp on what an API is and what it is used for. We will now begin showing a step-by-step process of how to create an API call that retrieves information from the BreweryDB library. BreweryDB is a fully moderated, open library of breweries and beers from all around the world. It keeps track of information such as location, website, phone numbers, descriptions, and logos. We will use this BreweryDB API to fetch all kinds of information from their library.
At the end of this how-to-guide you should be able to:

  • Acquire your own personnel API key.
  • Use that key to practice making API calls to the BreweryDB library.
  • Create a simple webpage that a user can enter information into a form.
  • Then display the desired information in a user-friendly way.
